接口说明¶

本章节为**STAYTRUE系列蓝牙模组**对接**飞利浦健身设备**的专属硬件接口、UART串口通信协议、功能指令手册，是健身设备（跑步机/单车/划船机等）与蓝牙模组开发对接、调试的核心技术依据，协议内容完全适配健身设备的蓝牙数据交互、状态上报、指令下发场景。

一、硬件接口对接规范¶

### 1.1 健身设备通用对接要求 1. 供电匹配：健身设备主板需为模组提供稳定直流供电，严格遵循各STAYTRUE模组VBAT电压范围，建议电源端并联100nF+10uF滤波电容，减少电机/电路干扰 2. 电平匹配：默认3.3V TTL电平，若健身设备MCU为5V，仅可通过SDC-LE-A3/SDC-LE-A3L的PA0/PA1高压IO对接，其余IO需加电平转换芯片 3. 接地规范：模组GND与健身设备主板电源地、信号地可靠共地，射频模块区域单独接地，避免电磁干扰影响蓝牙通信 4. 射频防护：板载天线区域远离健身设备金属部件、电机驱动电路，预留≥5mm净空区，保证蓝牙接收/发射性能 5. 串口接线：UART_RX/TX采用差分接线（若设备支持），普通接线需加屏蔽线，两端接100Ω匹配电阻，减少信号衰减

### 1.2 健身设备主流模组对接重点 #### SDC-LE-01（跑步机/单车/划船机主力适配） - 核心对接引脚：UART_RX（PD2）、UART_TX（PD3）、VCC（3.3V）、GND、RST - 复位脚处理：RST引脚可接健身设备MCU GPIO，异常时触发复位，正常工作时置为悬浮输入降低功耗 - 串口优先级：唯一通信串口，需独占用于健身设备状态上报与指令接收，禁止复用其他功能

#### SDC-LE-05（带音频健身设备适配，如智能动感单车） - 通信串口：P3（Tx/PE7）、P4（Rx/PE6），单独用于蓝牙协议交互，与音频功能引脚完全隔离 - 音频引脚：DACR（右声道）、DACL（左声道）接健身设备功放，MICIN接麦克风，适配语音播报/蓝牙音乐功能 - MUTE引脚：P5（PE0）可接健身设备静音按键，高电平触发全局音频静音，适配设备操作逻辑

#### SDC-LE-A3/SDC-LE-A3L（高端多功能健身设备适配） - 串口选择：可通过CrossBar IO映射，将UART分配至靠近健身设备MCU的引脚，减少接线长度 - 按键扩展：利用44按键矩阵扫描功能，对接健身设备的速度调节、模式切换等物理按键 - 状态指示：映射GPIO至健身设备蓝牙指示灯，实现蓝牙连接/配对/工作状态可视化

二、UART串口通信协议（健身设备专用）¶

## 2.1 固定串口参数（飞利浦健身设备协议标准）

  • 波特率：9600 bps（全系列模组适配，不可修改）

  • 数据位：8位

  • 停止位：1位

  • 校验位：无

  • 流控：无

  • 通信方向：健身设备MCU → 模组（指令下发）、模组 → 健身设备MCU（状态上报/响应）

  • 传输方式：字节流传输，十六进制格式，低位先行

## 2.2 统一数据帧格式（指令帧/上报帧通用） 帧结构总长度：N+6 字节**（N为数据段长度，0≤N≤16，适配健身设备短数据交互需求） 帧头 (2 字节) | 设备类型码 (1 字节) | 数据长度 (1 字节) | 数据段 (N 字节) | 校验位 (1 字节) | 帧尾 (1 字节) #### 各字段定义 1. **帧头：固定为 0xAA 0x55，用于帧起始识别，健身设备与模组双向通信统一帧头 2. 设备类型码：标识健身设备类型，飞利浦协议专属定义，不可自定义

  • 0x01：跑步机

  • 0x02：动感单车

  • 0x03：划船机

  • 0x04：椭圆机

  • 0x05：踏步机

  1. 数据长度：标识后续**数据段**的字节数，取值0~16，若无线数据段则为0x00

  2. 数据段：健身设备核心交互数据，如运动速度、阻力等级、指令参数等，按设备类型定义具体格式

  3. 校验位：异或校验，计算范围为「设备类型码 + 数据长度 + 数据段所有字节」，单字节校验值

  4. 帧尾：固定为 0x0D，用于帧结束识别

#### 校验位计算示例 设备类型码0x01（跑步机）、数据长度0x02、数据段0x05 0x10 校验位 = 0x01 ^ 0x02 ^ 0x05 ^ 0x10 = 0x16

## 2.3 通信交互规则（飞利浦健身设备协议规范） 1. 指令下发：健身设备MCU主动发送指令帧，模组接收并校验，100ms内**返回响应帧，无响应则MCU重发（最多3次） 2. **状态上报：模组按**固定周期**向健身设备MCU上报蓝牙状态（默认1s/次）；健身设备运动状态由MCU主动上报至模组，模组透传至蓝牙端（APP/终端） 3. 帧过滤：模组/MCU接收到帧头、帧尾不匹配，或校验位错误的帧，直接丢弃，不返回响应 4. 数据优先级：健身设备**急停指令**（0x00）为最高优先级，模组接收到后立即停止蓝牙数据透传，优先执行指令 5. 波特率容错：模组支持±5%波特率偏差，适配健身设备MCU时钟误差

三、核心功能指令集（飞利浦健身设备专属）¶

按**指令下发（MCU→模组）**和**状态上报（模组→MCU）**分类，所有指令均遵循统一帧格式，数据段为指令核心参数，适配飞利浦健身设备与Kinomap、Zwift等第三方APP的对接需求。

## 3.1 指令下发（健身设备MCU → STAYTRUE模组） 指令码为数据段**第1字节**，为飞利浦协议固定定义，模组接收到后执行对应功能并返回响应（响应帧数据段为指令码+执行状态：0x00成功/0x01失败）。

## 3.3 运动数据透传协议（健身设备 → 模组 → APP） 健身设备的核心运动数据由MCU按**500ms周期**上报至模组，模组透传至已连接的第三方APP（Kinomap/Zwift），数据段为固定格式，适配健身设备运动数据标准。 #### 通用运动数据段格式（8字节） 速度 (1 字节) | 距离 (2 字节) | 时间 (2 字节) | 心率 (1 字节) | 卡路里 (1 字节) | 坡度 / 阻力 (1 字节) #### 数据取值范围（飞利浦健身设备标准） - 速度：0~255（单位：0.1km/h，如0x10对应1.0km/h） - 距离：0~65535（单位：m，累计值） - 时间：0~65535（单位：s，运动时长） - 心率：0~255（单位：bpm，0表示未检测） - 卡路里：0~255（单位：kcal，累计值） - 坡度/阻力：0~100（跑步机为坡度，单车/划船机为阻力）

四、错误码定义（飞利浦协议标准）¶

所有错误码通过**响应帧/异常上报帧**返回，健身设备MCU可根据错误码定位问题，执行对应的故障处理逻辑（如重新发送指令、触发模组复位）。

五、开发调试与生产测试规范¶

## 5.1 开发调试重点 1. 调试串口复用：SDC-LE-05/P12、SDC-LE-A3L/P5为默认Log口，可通过该串口查看模组运行日志，定位蓝牙连接、数据透传问题 2. 运动数据模拟：开发阶段可通过MCU模拟发送运动数据，测试模组透传至APP的准确性，验证数据格式是否匹配 3. 异常模拟测试：模拟串口断连、蓝牙断连、参数错误等场景，测试模组与健身设备的故障处理逻辑 4. 第三方APP对接：优先对接Kinomap/Zwift，验证运动数据实时性、指令响应速度，确保符合APP对接标准

## 5.2 生产测试规范（飞利浦健身设备量产要求） 1. 蓝牙射频测试：量产前测试模组发射功率、接收灵敏度，确保蓝牙通信距离≥10米（无遮挡） 2. 串口通信测试：连续发送1000帧指令/数据，校验通信成功率≥99.9%，无丢帧、错帧 3. 运动数据校准：校准健身设备运动数据与模组透传数据的一致性，误差≤±1% 4. 高低温测试：在-40℃~85℃环境下，测试模组蓝牙通信、串口交互的稳定性，无功能异常 5. 老化测试：模组连续工作72小时，测试蓝牙连接、数据透传的持续性，无死机、断连

## 5.3 常见问题排查 1. 蓝牙无法发现：检查模组供电是否正常→确认蓝牙开启指令已发送→检查射频天线区域是否被遮挡→复位模组 2. 串口无响应：核对串口参数（9600-8-N-1）→检查RX/TX接线是否反接→测试模组硬件是否正常 3. 数据透传延迟：缩短运动数据上报周期→优化健身设备主板电路，减少电磁干扰→检查蓝牙连接信号强度 4. APP无法接收数据：核对运动数据段格式→确认设备类型码正确→检查模组与APP的配对连接状态
